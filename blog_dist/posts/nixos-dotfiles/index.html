Managing dotfiles with Nix (on NixOS)

Posted on 2022-12-27 16:36:11 -0500 EST

This tutorial will be using Flake, and HomeManager.

Based on my dotfiles

Step 1: Enabling nix flakes in /etc/nixos/configuration.nix

Flakes are not enabled by default on NixOS, so you need to enable them. We can do this by editing /etc/nixos/configuration.nix.

At the top of the file add the following lines:

nix.package = pkgs.nixVersions.stable; # Use the latest stable version of Nix
nix.extraOptions = ''
    experimental-features = nix-command flakes
''; # Enables nix flakes, and the nix command

Step 2: Creating a flake.nix

Here we are going to create a flake.nix file which will output system

configurations, and home-manager profiles, and take in home-manager, and nixpkgs as inputs. In the folder you want to store your dotfiles create

a file called flake.nix and add the following content (COMMENTS EXPLAIN THE CODE)

{
  description = "Home manager config";

  # Pull in nixos unstable, and home manager as inputs
  inputs = {
    # url to nixpkgs unstable
    nixpkgs.url = "github:nixos/nixpkgs/nixos-unstable";

    # url to home-manager which uses nixpkgs
    home-manager = {
      url = "github:nix-community/home-manager";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = {
    self,
    nixpkgs,
    home-manager,
    ...
  }: let
    system = "x86_64-linux";
    pkgs = import nixpkgs {
      inherit system;
      
      # Uncomment the following line to enable unfree packages
      # config = {allowUnfree = true;};
    };
    # You can change this hostname to be whatever you want
    hostname = "celeste";
    lib = nixpkgs.lib;
  in {

    # Create a home manager profile as an output(based on hostname)
    homeConfigurations.${hostname} = home-manager.lib.homeManagerConfiguration {
      inherit pkgs; # Inherit packages

      modules = [
        hyprland.homeManagerModules.default # Default home-manager module

        # Path to home manager config
        ./hosts/celeste/user/home.nix
      ];
    };

    # Create a nixos configuration as an output
    nixosConfigurations.${hostname} = nixpkgs.lib.nixosSystem {
      inherit system;
      modules = [
        # This file can be copied from /etc/nixos/configuration.nxix
        ./hosts/celeste/system/configuration.nix
      ];
    };

    # Activation scripts
    apps.${system} = {
        
      # Create an activation script for our home-manager derivation that was  created
      "activate/${hostname}" = let
        drv = self.outputs.homeConfigurations.${hostname}.activationPackage;
        exePath = "/activate";
      in {
        type = "app";
        program = "${drv}${exePath}";
      };
    };
  };
}

Step 3: Creating folers

We still need to store our home-manager, and NixOS configuration somewhere! The flake pulls in these files from specific locations.

Below is a tree of our dotfiles repo after the folders are created

. # ~/.df or ~/.dotfiles, or where ever your dotfiles are located
├── bootstrap # Shell script to manage our config(created later)
├── flake.lock # Lock file generated by Nix(created later)
├── flake.nix # Flake config we just wrote
├── hosts # Folder with subfolders named after each host's name
│   └── celeste
│       ├── configs # Configs for programs, plain text files
│       ├── modules # Contains .nix files which can configure programs using home-manager
│       ├── system # Contents of /etc/nixos
│       │   ├── configuration.nix # NixOS configuration file
│       │   ├── hardware-configuration.nix # Auto-generated hardware config file
│       └── user # User specific configs
│           └── home.nix # Home-manager config
├── README.md # README for our dotfiles, not needed

Step 4: Creating a bootstrap script

Below is a small shell script which allows managing our dotfiles

#!/usr/bin/env bash

# NixOS bootstrap script
# Run this script on a clean install of NixOS to bootstrap it with my configs

OFF='\033[0m'       # Text Reset
RED='\033[0;31m'          # Red
BLUE='\033[0;34m'         # Blue

# Check for sudo, if doas is found use that
export AUTHPATH="$(which sudo)"

if [[ -f "$(which doas)" ]]; then
  export AUTHPATH="$(which doas)"
fi

# This function logs info to the command line
function log(){
  echo -e "$BLUE~>$OFF $1"
}

# This runs the home-manager activate script for our host we created in flake.nix
function homeManagerSetup(){
	nix run '.#activate/celeste'
}

# Updates the nix flake lock file with the latest inputs
log "Updating Nix flake lock file..."
nix flake update

# Only build the home-manager config
if [[ "$1" == "--only-home-manager" ]]; then 
  log "Building home-manager config flake..."
  homeManagerSetup
  exit 0
fi

# Allows changing of the nixos-rebuild command line arguments
export REBUILD_FLAGS=""
if [[ "$1" == "--update-grub" ]]; then
  export REBUILD_FLAGS="--install-bootloader"
  log "Notice: System update will also update GRUB"
fi

log "Applying system config..."
# Execute the nixos-rebuild switch command using the current flake
# (The rebuild command will use the config for our current hostname)
# with root privileges
command $AUTHPATH nixos-rebuild switch --flake .# $REBUILD_FLAGS
# Run home-manager setup if we have not already
log "Building home-manager config flake..."
homeManagerSetup

This bootstrap script will allow us to install our home-manager configuration, and home-manager config without having to manually run nix commands.

Step 5: Setting up home manager

We need to create a home-manager config file for our setup to be complete, first create a file located in hosts/[HOSTNAME]/user/home.nix, then add the following text

{
  config,
  pkgs,
  lib,
  ...
}: {
  imports = [
    ../modules # Import ../modules/default.nix
  ];
  home.username = "interfiber"; # Username goes here
  home.homeDirectory = "/home/interfiber"; # Home directory goes here
  home.stateVersion = "22.11"; # Dont change this
  programs.home-manager.enable = true; # Home manager manages home manager
}

Now, this will try to import default.nix from our modules folder, which is empty right now. Which we will now fix First create a file located at hosts/[HOSTNAME]/modules/default.nix, this is imported by default when importing a folder Now add the following text to it

{self, ...}: {
  imports = [
    # List all of the modules you want to import here, for example:
    # ./dunst.nix would manage the dunst notification daemon, and so on
  ];
}

By reading the comments we can see that this file imports all of the modules inside the modules folder.

Each module will manage its own program or set of programs.

Step 6: Adding modules

Before I end this tutorial I will show you how to create a basic module, which will manage the kitty terminal. First I will create hosts/[HOSTNAME]/modules/kitty.nix, and add the following content

{pkgs, ...}:
with builtins; {
  programs.kitty = {
    # Enable the kitty program in our profile, which will install it
    enable = true;
    
    # Instead of having our kitty.conf file in our dotfiles repo, and updating it ourselves
    # we can have home manager create, and update it for us like below.

    # This is a attribute set of (string or boolean or signed integer), the contents of this list will be converted into the contents of our kitty.conf file
    # The offical kitty configuration options are located here: https://sw.kovidgoyal.net/kitty/conf

    settings = {
      font_family = "JetBrainsMono Nerd Font Mono";
      font_size = 30;
      cursor_shape = "Beam";
      background = "#0d0f18";
      foreground = "#a5b6cf";
      window_padding_width = 5;
    };
  };
}

Now make note that if you are managing your dotfiles with git, that nix will only copy the currently tracked files

into the nix store, so make sure that you commit newly created files to prevent errors. Now time to add this imports to our modules/default.nix file, it should now look like this:

{self, ...}: {
  imports = [
    ./kitty.nix
  ];
}

And now we have added the module! Great now lets run the boostrap script like below to test if everything worked

chmod +x ./bootstrap

# ./bootstrap --only-home-manager if you only want to update home-manager
# ./bootstrap if you want to update everything
./boostrap

And if we run cat ~/.config/kitty/kitty.conf we should see that home manager created, and populated it for us!
